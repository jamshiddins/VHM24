Отчет вендинг-системы

Описание отчета

Основной источник данных о продажах по всем автоматам. Файл формата Excel (OrderStatisticTable*.xlsx, лист sheet1).

Используемые поля:

  • Order number — уникальный ID каждой продажи. Используется как ключ для связывания с другими системами.
  • Creation time — точное время заказа. Служит для определения периода, и как дополнительный ключ при сверке.
  • Order resource — тип оплаты: Cash payment, Pay card, Custom payment.
  • Order status — должен быть Paid. Прочее отбрасывается.
  • Brew status — должен быть Delivered. Если Failed, фиксируется как сбой доставки.
  • Order price — сумма заказа в UZS. Основной числовой показатель.
  • Machine Code — уникальный код автомата. Используется для фильтрации, аналитики, связки с инкассацией.
  • Goods name, Flavour name — используются для аналитики по продуктам (не обязательны для финсверки).

Особенности: Все оплаты Custom payment — не уточнены, и требуют декодирования из шлюза. Некоторые поля могут быть пустыми, нужно предусмотреть fallback.

Отчет платежного шлюза

Описание отчета

Расшифровка Custom payment и дополнение платежей по QR/Payme/Click/Uzum. Файл формата Excel (report (1).xlsx, лист in).

Используемые поля:

  • Order number — используется для связывания с заказом из вендинг-отчёта.
  • Time — время платежа. Используется для дополнительной сверки с Creation time.
  • Goods name — повтор товара.
  • Goods ID — внутренний ID товара, используется в чеке, важен для фискализации.
  • Payment type — точный платёжный провайдер: Payme, Click, Uzum, VIP, Cash, Card. Заменяет значение Custom payment из вендинг-отчёта.
  • Machine Code — дополнительная проверка.

Особенности: Может включать больше продаж, чем в вендинг-отчёте (нефискализированные вручную). Некоторые оплаты по VIP/тесту идут вне фискализации — их нужно уметь распознавать.

Сводный отчет

Описание отчета

Сводный отчет объединяет данные из всех источников и выявляет несоответствия. Для формирования отчета необходимо загрузить все исходные отчеты.

Отчет позволяет:

  • Сверить данные о продажах из разных источников
  • Выявить ошибки и несоответствия
  • Проверить фискализацию продаж
  • Проверить поступления в банк
  • Проверить инкассацию наличных
